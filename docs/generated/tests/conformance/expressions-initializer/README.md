## Claims

### Initializer Expressions

- **C1** — When the base expression of a call is a type instead of a value, the expression is an initializer expression that creates an instance of the named type. (The doc's example: `float2(1.0f, 2.0f)`.)
- **C2** — An initializer expression initializes an instance of the specified type using the given arguments (argument values flow into the instance's fields/components).
- **C3** — An initializer expression with only a single argument is treated as a cast expression; `int(1.0f)` and `(int) 1.0f` are equivalent.

### Initializer List Expression

- **C4** — An initializer list expression comprises zero or more expressions, separated by commas, enclosed in `{}`.
- **C5** — An initializer list expression may only be used directly as the initial-value expression of a variable or parameter declaration; initializer lists are not allowed as arbitrary sub-expressions.
