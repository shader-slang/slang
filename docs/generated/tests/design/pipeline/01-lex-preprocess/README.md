# Tests for pipeline/01-lex-preprocess

## Intent

Tests verify the lex-and-preprocess pipeline behaviors described in
[`docs/generated/design/pipeline/01-lex-preprocess.md`](../../../../design/pipeline/01-lex-preprocess.md):
the lexer's special-case handling (backslash continuation), literal
scanning and value extraction (escape decoding, scan-time vs
decode-time diagnostics, float overflow/underflow), macro expansion
(object-like, function-like, parameter-op replay, `#`, `##`,
`__LINE__`, `__FILE__`), conditional compilation
(`#if`/`#ifdef`/`#elif`/`#else`/`#endif`/`#undef`), `#include`
resolution (quoted and angle forms), `#pragma warning` state
tracking, and the failure modes called out by the `## Failure
modes` section.

Coverage strategy: at least one functional test per substantive
positive claim, boundary probes along the numeric/escape axes the
doc names, and at least one diagnostic test per "is rejected"-style
claim with the diagnostic code pinned. Pure lex-surface claims that
`syntax-reference/tokens` already exercises (numeric suffixes,
comment shapes, raw strings) are intentionally NOT duplicated
here.

## Boundary / stress coverage added in expansion

The expansion pass added boundary, stress, and negative probes
against the axes called out in `_common.md` (Pressure the
compiler):

- **`#if` expression**: 0 (lower edge), `0xFFFFFFFF` (uint32
  MAX), `2147483647` (int32 MAX), signed arithmetic overflow
  (`INT_MAX + 1` wraps), undefined identifier treated as 0,
  five-deep nesting, five-arm `#elif` chain.
- **Macro expansion**: zero-parameter macro, eight-parameter
  macro (many-args stress), empty argument substitution,
  wrong-argument-count diagnostic (negative), argument-is-a-macro
  expansion, stringized-argument non-expansion.
- **Stringize / paste**: empty argument to `#x` yields `""`,
  `##` with empty right-hand operand collapses to the left
  operand, `##` of two numeric literals re-lexes to one literal.
- **`#include`**: five-deep chain, self-include cycle (negative
  â€” cyclic include diagnostic), header-guard pattern via
  `#ifndef` / `#define`, escape sequences in include paths kept
  verbatim.
- **Predefined macros**: `__LINE__` after `#line N` reports a
  shifted value (far-line boundary); `__LINE__` through nested
  expansion reports the top-level line. `__FILE__` boundary is
  covered by the existing functional test.
- **Line continuation**: backslash-newline inside a string
  literal (in addition to the existing in-macro test).
- **`#error` / `#warning`**: empty-message edge, long-message
  edge, macro-name-in-message (verbatim, not expanded).
- **`#pragma`**: unknown pragma is a warning (not an error);
  unknown pragma with arguments still warns; `#pragma once` in
  a doubly-included header prevents duplicate declarations;
  `#pragma warning` disable timeline and push/pop restore.
- **Literal escapes**: every documented escape family (octal,
  hex, `\u`, `\U`, braced) at a known code point; braced form
  above the BMP (32-bit edge); hex escape with zero digits
  (negative), more digits than one byte allows in a string
  (decode-time negative), and a code point UTF-8 cannot encode
  (decode-time negative); empty and multi-character character
  literals (scan-time negatives); newline in a literal
  (scan-time negative).
- **Float literal range**: overflow parses as INFINITY,
  underflow parses as 0.

Skipped boundary axes:

- **Recursive macro definition** (negative) â€” the doc does not
  commit to detection semantics; recorded as a doc gap above.
- **Line continuation in `#include` path** â€” Slang does not
  document line-continuation behavior inside an include path
  specifically; no observable claim to anchor to.
- **Malformed UTF-8 bytes in literals** â€” needs byte-level file
  generation; recorded under Untested claims.
