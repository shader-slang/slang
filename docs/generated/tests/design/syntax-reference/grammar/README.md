# Tests for syntax-reference/grammar

## Intent

Tests verify the surface-grammar productions described in
[`docs/generated/design/syntax-reference/grammar.md`](../../../design/syntax-reference/grammar.md):
top-level/module structure, declaration / expression / statement /
type / generic / modifier productions, the precedence ladder (now
including `is` / `as` at level 6), the literal-forms subsection, the
repeated-keyword where-clause shape with its pack / coercion terms,
and the `<` disambiguation fork. Most tests are `pipeline_stage=parse`
and use `//TEST:INTERPRET` so the production shape is observed
end-to-end on the interpreter without a GPU; claims involving
exceptions (`throws` / `do ... catch`), existential `is` / `as`,
pointers, and `Optional` use `//TEST:COMPARE_COMPUTE ... -cpu` because
those features are unreliable or unsupported under INTERPRET. Four
negative tests use `//DIAGNOSTIC_TEST:SIMPLE` to assert parse-stage
rejection with a pinned E-code.

This bundle is intentionally disjoint from
[`syntax-reference/tokens`](../tokens/) (which is lexer-level) and
from `pipeline/02-parse-ast` (which is parser-stage internal
behavior). The angle here is **production-shape**: which strings the
grammar names, and in what shape they parse.
